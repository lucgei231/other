Lesson 16: Asynchronous JavaScript

In this lesson, we will explore asynchronous programming in JavaScript, focusing on callbacks, promises, and async/await.

1. Callbacks

Callbacks are functions that are passed as arguments to other functions and are executed after a certain event occurs.


function fetchData(callback) {
    setTimeout(() => {
        const data = { name: 'John', age: 30 };
        callback(data);
    }, 1000);
}

fetchData((data) => {
    console.log(data);
});

2. Promises

Promises are objects that represent the eventual completion (or failure) of an asynchronous operation and its resulting value.


function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const data = { name: 'Jane', age: 25 };
            resolve(data);
        }, 1000);
    });
}

fetchData().then(data => {
    console.log(data);
});

3. Async/Await

Async/await is a syntactic sugar built on top of promises, making asynchronous code easier to read and write.


async function fetchData() {
    const data = await new Promise((resolve) => {
        setTimeout(() => {
            resolve({ name: 'Doe', age: 28 });
        }, 1000);
    });
    console.log(data);
}

fetchData();

Exercises

Try to implement the following:

  • Create a function that fetches data using a callback.
  • Rewrite the callback function using promises.
  • Use async/await to fetch data and log it to the console.

Conclusion

Asynchronous programming is a powerful feature in JavaScript that allows you to handle operations that take time, such as network requests, without blocking the main thread.
