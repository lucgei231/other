Lesson 06: Understanding Asynchronous JavaScript

In this lesson, we will explore asynchronous programming in JavaScript, focusing on callbacks, promises, and async/await.

1. Callbacks

Callbacks are functions that are passed as arguments to other functions and are executed after a certain event occurs.


function fetchData(callback) {
    setTimeout(() => {
        const data = { name: 'John', age: 30 };
        callback(data);
    }, 2000);
}

fetchData((data) => {
    console.log(data);
});

2. Promises

Promises are objects that represent the eventual completion (or failure) of an asynchronous operation and its resulting value.


function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const data = { name: 'Jane', age: 25 };
            resolve(data);
        }, 2000);
    });
}

fetchData().then(data => {
    console.log(data);
});

3. Async/Await

Async/await is a syntactic sugar built on top of promises, making asynchronous code easier to read and write.


async function fetchData() {
    const data = await new Promise((resolve) => {
        setTimeout(() => {
            resolve({ name: 'Doe', age: 28 });
        }, 2000);
    });
    console.log(data);
}

fetchData();

Exercises

Try to implement your own version of the fetchData function using both callbacks and promises.

Conclusion

Asynchronous programming is a powerful feature in JavaScript that allows you to handle operations that take time without blocking the execution of your code.
